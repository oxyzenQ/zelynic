## First Real Write

The first real filesystem write in Zelynic's history occurred during phase
2b. The write was `mkdir`-only under `/sys/fs/cgroup/zelynic/`.

### What Was Written

- Created `/sys/fs/cgroup/zelynic` (parent namespace directory) if absent.
- Created `/sys/fs/cgroup/zelynic/target_sleep` (target cgroup directory).
- Verified the target directory exists after creation.
- Cleaned up the target directory only if empty and operation-owned.
- Left `/sys/fs/cgroup/zelynic` in place (parent namespace may remain).

### What Was NOT Written

- No PID was moved into any Zelynic target cgroup.
- No `cgroup.procs` write was performed in any cgroup.
- No limiter attach was performed.
- No nftables rules were added or modified.
- No tc qdisc, class, or filter state was changed.
- No Zelynic state file (`/run/zelynic/state.json`) was written.
- No systemd unit modification beyond the probe scope.
- No modification to any cgroup outside `/sys/fs/cgroup/zelynic/`.

### Cleanup

- The target cgroup `target_sleep` was created, verified, then removed
  (empty, operation-owned cleanup).
- The parent namespace `/sys/fs/cgroup/zelynic` was left in place.
- The transient systemd scope transitioned to inactive/dead after the child
  process exited.

## Output Honesty Fix (Phase 2b.1)

The canonical safety footer previously read:

> "No nftables, tc, Zelynic cgroup, or state changes were made."

This was misleading when `--mkdir-live` was active, because mkdir-live
intentionally creates cgroup directories. The footer was replaced with
truthful wording for the mkdir-live path:

- "No nftables, tc, or Zelynic state changes were made."
- "Mkdir-only cgroup preparation was performed."

Additional honest lines were added to the mkdir experiment section output:

- "No cgroup.procs write was performed."
- "Parent namespace may remain: /sys/fs/cgroup/zelynic"

The error message for the `--mkdir-live` + `--attach-live` combination was
also made honest:

- "Mkdir-only experiment completed; experimental PID move is not
  implemented yet."

This replaces the generic `EXPERIMENTAL_PID_MOVE_NOT_IMPLEMENTED` when
mkdir-live is active, making clear that the mkdir experiment completed
successfully while PID move remains unimplemented.

## Root mkdir-live Smoke Validation

When run with the full experimental consent bundle plus `--mkdir-live` as root,
the command:

1. Launches a transient systemd scope via `systemd-run --scope`.
2. Discovers ControlGroup and PID from `cgroup.procs`.
3. Captures original cgroup from `/proc/<pid>/cgroup`.
4. Runs existing read-only safety checks (liveness, self-protection).
5. Creates `/sys/fs/cgroup/zelynic` (namespace directory).
6. Creates `/sys/fs/cgroup/zelynic/target_<name>` (target cgroup).
7. Verifies the target cgroup directory exists.
8. Removes the target cgroup (empty, operation-owned cleanup).
9. Leaves `/sys/fs/cgroup/zelynic` in place.
10. Prints truthful mkdir-live safety footer.
11. Returns "Mkdir-only experiment completed; experimental PID move is not
    implemented yet."
12. Transient scope ends inactive/dead.

**No mutation beyond mkdir was observed:**

- No PID was moved into any Zelynic target cgroup.
- No limiter attach was performed.
- No nftables rules were added or modified.
- No tc/qdisc/filter state was changed.
- No Zelynic state files were written.
- Bandwidth limiting is not active from this command.

## Next Milestone: Phase 3 (Design-Only)

Phase 3 is the next planned phase and will introduce design-only work for the
single PID move-only experiment with immediate rollback. Phase 3 does not
implement PID movement; it produces design documentation for the move sequence,
rollback protocol, and verification steps.

### Phase 3 Constraints

The first actual PID move (when eventually implemented in a future phase)
must satisfy all of the following constraints:

- Root-only, system-scope-only.
- Single disposable PID only (e.g., `sleep 3` or `sleep 10`).
- Immediate rollback: PID visits the target cgroup and returns to its original
  cgroup within the same operation.
- No limiter attach (no nftables, no tc, no bandwidth enforcement).
- No nftables/tc/state mutation.
- No persistent state write (`/run/zelynic/state.json` is not written).
- All safety gates re-evaluated immediately before the write moment.
- PID liveness rechecked at move time (PID may exit between discovery and move).
- Original cgroup path captured and validated before move.
- Rollback failure reported honestly; no silent ignore.
- Target cgroup removed only if empty and operation-owned.

These constraints are already documented in
[docs/experimental-pid-move-lab.md](experimental-pid-move-lab.md) under the
"PID Move-Only Boundary" and "Rollback Rules" sections.
