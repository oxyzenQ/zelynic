# v3.1 Phase 2: Pure App Identity Model

## Status

**Phase 2 — Pure model + deterministic unit tests. No CLI wiring. No live process scanning.**

## Source of Truth

`docs/v3.1-phase-1-app-identity-network-ledger-contract.md` (phase 1 contract design)

## Summary

Phase 2 implements the pure Rust identity model types defined in the phase 1
contract. These are data-only types with serde serialization — no live
filesystem reads, no process scanning, no CLI wiring, no persistence.

## Model Types Added

| Type | Module | Description |
|------|--------|-------------|
| `UsageAttributionScope` | `src/accounting/identity.rs` | Enum: `InterfaceOnly`, `ProcessBestEffort`, `CgroupBestEffort`, `TargetBestEffort`, `Unknown`. Serializes as snake_case via serde. |
| `ProcessIdentity` | `src/accounting/identity.rs` | Struct with optional fields: `pid`, `comm`, `argv0`, `cmdline`, `executable_path`. All fields use `skip_serializing_if = "Option::is_none"`. |
| `CgroupIdentity` | `src/accounting/identity.rs` | Struct with optional fields: `cgroup_path`, `systemd_unit`, `systemd_scope`. All fields use `skip_serializing_if`. |
| `InterfaceIdentity` | `src/accounting/identity.rs` | Struct: `name` (String), `loopback` (bool). Constructor `new(name, loopback)` and static `is_loopback_name(name)`. |
| `TargetIdentity` | `src/accounting/identity.rs` | Composed struct: optional `process`, `cgroup`, `interface`. Constructors: `empty()`, `interface_only()`. |
| `ResolvedUsageTarget` | `src/accounting/identity.rs` | Pairs `TargetIdentity` with `UsageAttributionScope`. Constructors: `unknown()`, `interface_only()`, `new()`. |
| `IdentityHonesty` | `src/accounting/identity.rs` | Honesty flags: `attribution_is_best_effort`, `interface_level_data_source`, `per_app_attribution_may_be_partial`, `enforcement_active` (always false), `persistence_performed` (always false). |
| `default_identity_honesty()` | `src/accounting/identity.rs` | Returns default honesty flags. Attribution=true, enforcement=false, persistence=false. |
| `render_resolved_target()` | `src/accounting/identity.rs` | Human-readable render of a resolved target. Includes attribution scope, identity fields, and best-effort disclaimers. |
| `serialize_identity_json()` | `src/accounting/identity.rs` | Deterministic pretty JSON serialization of `ResolvedUsageTarget`. |
| `deserialize_identity_json()` | `src/accounting/identity.rs` | Deserialization for round-trip testing. |
| `serialize_identity_honesty()` | `src/accounting/identity.rs` | JSON serialization of `IdentityHonesty`. |

## What This Phase Does NOT Implement

- No live process scanning or `/proc` reads.
- No CLI command wiring (no `ledger` or `identity` subcommand).
- No persistence or file writes.
- No enforcement, blocking, or quota.
- No eBPF, nft/tc mutation, cgroup mutation, PID movement.
- No change to existing `usage --sample` behavior.
- No change to existing JSON schema (v3.0 schema_version=1 unchanged).
- No version bump.
