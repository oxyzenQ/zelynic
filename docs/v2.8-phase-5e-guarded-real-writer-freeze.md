# Zelynic v2.8 Phase 5e — Guarded Real Writer Seam Freeze / Non-Exposure Audit

## Overview

This document freezes the guarded real writer seam and audits its non-exposure
across the Zelynic codebase. Phase 5e is **docs/report-only**: it produces this
freeze report and updates related documentation but introduces **no runtime
changes, no PID movement, no `cgroup.procs` writes, no limiter attach, and no
state mutation**.

The freeze report summarizes all phase 5 work (5a through 5d), documents the
current test state, confirms the seam's hard-blocked status, audits that no CLI
or runtime path reaches the guarded real writer, and defines the entry criteria
that must be satisfied before any future phase performs a real `cgroup.procs`
write.

## Phase 5e Boundary

### What Phase 5e Produces

- This freeze/non-exposure audit report
  (`docs/v2.8-phase-5e-guarded-real-writer-freeze.md`).
- Updates to `docs/experimental-pid-move-lab.md` with phase 5d completed and
  phase 5e current/freeze status.
- Updates to `CHANGELOG.md` with a phase 5e entry.
- No Rust code changes. No runtime behavior changes. No new CLI flags.

### What Phase 5e Does NOT Do

- Does NOT implement live PID movement.
- Does NOT write `cgroup.procs`.
- Does NOT add runtime mutation of any kind.
- Does NOT add limiter attach, nftables/tc enforcement, or state writes.
- Does NOT change CLI behavior.
- Does NOT modify any Rust source file (except optional low-impact tests).
- Does NOT enable any code path for live PID move.

## Explicit Seam Freeze Guarantees

The guarded real writer seam (`src/systemd_wrapper/guarded_real_writer.rs`)
provides the following freeze guarantees. These guarantees are invariant: they
must remain true for all future phases until a new freeze report explicitly
relaxes them.

### Hard-Blocked Status

| Guarantee | Value | Enforcement |
|-----------|-------|-------------|
| Seam always returns blocked/not implemented | True | `build_guarded_real_writer_plan()` always returns `status: "blocked"` |
| Seam never performs any write operation | True | No I/O, no filesystem access, no /proc, no /sys |
| Seam result fields are hardcoded non-mutating | True | `pid_location: NotMoved`, all mutation flags `false` |

### Non-Mutation Guarantees

1. **No live PID move**: The seam never moves any PID. The `pid_location` field
   is always `NotMoved`. No `cgroup.procs` file is read or written.

2. **No real cgroup.procs write**: The `cgroup_procs_writes_performed` field is
   always `false`. The seam contains no filesystem write operations.

3. **No rollback write**: The `rollback_attempted` field is always `false`. No
   rollback cgroup.procs write is ever performed.

4. **No cleanup mutation**: The `cleanup_attempted` field is always `false`. No
   directory creation or removal is performed.

5. **No limiter attach**: The `limiter_attach_performed` field is always `false`.
   The seam contains no limiter-related code.

6. **No nftables/tc/Zelynic state mutation**: The
   `nft_tc_state_mutation_performed` field is always `false`. The seam contains
   no nftables, tc, or state-related code.

7. **No persistent state write**: One of the 7 canonical deny lines explicitly
   states "No persistent state write was performed." in every output.

8. **No CLI path enabled**: One of the 7 canonical deny lines explicitly
   states "No CLI path for live PID move was enabled." in every output.

9. **No /proc access**: The seam performs no reads from `/proc/<pid>/cgroup`,
   `/proc/<pid>/status`, or any other `/proc` file.

10. **No /sys access**: The seam performs no reads from or writes to any path
    under `/sys/fs/cgroup` or `/sys/`.

11. **No filesystem mutation**: The seam performs no `mkdir`, `rmdir`, `write`,
    `create`, `remove`, or any other filesystem mutation operation. It is pure
    computation: takes an input struct, returns a result struct.

## Non-Exposure Audit

This section audits the guarded real writer seam's non-exposure across the
Zelynic codebase to confirm that no CLI command, runtime path, or public
interface reaches the guarded real writer seam.

### Module Registration

The `guarded_real_writer` module is registered in
`src/systemd_wrapper/mod.rs` as:

```rust

```

It is a crate-private module (`pub(crate)` items only). It is not re-exported
as a public item. No external crate can access it.

## Future Phase Entry Criteria Before Any Real cgroup.procs Write

The following criteria must ALL be satisfied before any future phase performs
a real `cgroup.procs` write. No deviation from these criteria is permitted
without a new design phase that produces an explicit document relaxing the
constraints.

### Mandatory Conditions

| # | Criterion | Enforcement |
|---|-----------|-------------|
| 1 | **Root-only**: Operation must run as UID 0 with `CAP_SYS_ADMIN` | Runtime gate: `getuid() == 0` |
| 2 | **System-scope-only**: PID must be in systemd system scope | Runtime gate: cgroup path not under `/user.slice/` |
| 3 | **Single disposable sleep PID only**: Target must be `sleep N` with no children | Runtime gate: PID count == 1, PID liveness check |
| 4 | **Original cgroup captured and verified**: `/proc/<pid>/cgroup` read succeeds | Runtime gate: cgroup path present, non-empty, not Zelynic-managed |
| 5 | **Target cgroup under `/sys/fs/cgroup/zelynic/` only**: No path traversal | Runtime gate: `starts_with("/sys/fs/cgroup/zelynic/")` and no `..` |
| 6 | **Rollback consent present**: `--rollback-required` flag is set | CLI gate: Clap `requires` + runtime check |
| 7 | **Rollback path reviewed**: Original cgroup verified writable before move | Runtime gate: `test -w` on original `cgroup.procs` |
| 8 | **Manual recovery reviewed**: Operator has reviewed manual recovery checklist | Documentation gate: phase 5b operator checklist |
| 9 | **Exact smoke commands reviewed**: Operator has reviewed all 14 smoke steps | Documentation gate: phase 5a manual smoke plan |
| 10 | **No limiter/nft/tc/state**: No bandwidth limiting, no nftables, no tc, no state | Architecture gate: live writer has no limiter/nft/tc methods |
| 11 | **Explicit operator confirmation before real smoke**: Operator acknowledges risk | Process gate: operator confirmation step in checklist |

### Additional Constraints

- The live writer must implement only `CgroupProcsWriter` trait methods
  (write_pid, read_pids, mkdir, rmdir_if_empty).
- The live writer must have no fields or methods related to limiters, BPF
  programs, tc classes, nftables, tc configurations, or Zelynic persistent
  state.
- The live writer must only operate on paths under `/sys/fs/cgroup/zelynic/`
  (for mkdir/rmdir) or on explicitly provided cgroup.procs paths (for
  read/write).
- No retry loops for any write operation. Each cgroup.procs write is attempted
  at most once.
- No multi-PID writes. Each cgroup.procs write targets exactly one PID.
- No persistent PID placement. The PID must be restored to its original cgroup
  before the operation returns.
