# Zelynic v3.0 Phase 8b: `usage --sample --json` CLI Validation Freeze / Contract Audit

This document freezes and audits the first machine-readable CLI output (`zelynic
usage --sample --json`) before adding delta sampling, filters, persistence, or any
future automation. Phase 8b is a **validation/documentation** phase with optional
small tests that directly protect the JSON CLI contract. No new features are added.

## Purpose

Phase 8b serves as the JSON CLI contract freeze point. It summarizes the full
chain from JSON schema design (phase 6) through pure JSON model (phase 7), model
validation (phase 7b), and CLI wiring (phase 8), then audits the end-to-end
behavior to confirm the contract is correctly implemented. This freeze establishes
a trusted baseline that future phases (delta sampling, interface filtering,
persistence) must not violate.

## Phase History Summary

### Phase 6: JSON Output Contract Design

Designed the JSON output schema for `zelynic usage --sample --json`. Key decisions:

- **Schema version**: integer field (currently `1`) for forward compatibility.
- **Command field**: always `"usage --sample --json"` for output traceability.
- **Source fields**: `source_path = "/proc/net/dev"` (hardcoded), `source_label = "live_proc_net_dev"`.
- **sampled_at policy**: omitted when caller does not provide a timestamp. No silent
  wall-clock generation via `SystemTime::now()` or `Instant::now()`.
- **12 honesty boolean flags**: all constant in v3.0. `interface_level_only = true`, all
  mutation/enforcement flags `false`.
- **Error contract**: three error types (`read_error`, `parse_error`,
  `unsupported_flag_error`). All errors retain full honesty flags. Error object is
  mutually exclusive with interface data (empty interfaces, zero totals on error).
- **CLI behavior contract**: `--json` requires `--sample`. Single-shot only. No loop,
  no watch, no delta, no interval.
- **Warnings array**: always present with counter-reset and attribution warnings.
- Design-only: no Rust code changes, no CLI flag registration.

### Phase 7: Pure JSON Model + Serialization Tests

Implemented `src/accounting/usage_json.rs` (318 LOC) with:

- Model types: `UsageJsonOutput`, `UsageJsonInterface`, `UsageJsonTotals`,
  `UsageJsonHonesty`, `UsageJsonError`, `UsageJsonErrorType`.
- Pure functions: `build_usage_json_from_snapshot()`, `build_usage_json_error()`,
  `serialize_usage_json()`, `deserialize_usage_json()`, `default_honesty_flags()`,
  `default_warnings()`.
- 12 honesty boolean flags (all constant in v3.0).
- 3 error types with explicit `#[serde(rename = "...")]` attributes.
- `sampled_at` uses `#[serde(skip_serializing_if = "Option::is_none")]`.
- 41 tests in `src/accounting/tests/usage_json.rs` (602 LOC).
- No CLI flag registration. Model is `pub(crate)` only.

### Phase 7b: JSON Model Validation / Error-Type Contract Freeze

Validation/documentation-only phase confirming the JSON model before CLI wiring:

- Verified canonical error type strings: `read_error`, `parse_error`,
  `unsupported_flag_error` via explicit `#[serde(rename = "...")]` attributes.
- Confirmed all 12 honesty flags are constant in v3.0.
- Confirmed `sampled_at` policy: omitted when None, never silently generated.
- Multi-vector test proof: substring tests, round-trip tests, Display trait tests.
- No code changes. No new tests.

### Phase 8: Wire `--json` to `usage --sample` CLI

Wired the `--json` flag to the existing `usage --sample` command:

- Added `json: bool` flag to `Usage` variant in `src/cli.rs` with `requires = "sample"`.
- `handle_usage_sample(json: bool)` routes to text or JSON output.
- `build_json_from_plan()` maps `LiveProcNetDevReadPlan` to `UsageJsonOutput`:
  success uses `build_usage_json_from_snapshot()`, errors use `build_usage_json_error()`.
- `sampled_at` is `None` (omitted from JSON).
- JSON output printed to stdout only, no human text prefix/suffix.
- Single-shot: one read, one parse, one JSON output, exit.
- Text output (`usage --sample`) unchanged — all 13 honesty lines remain.
- 13 new tests: CLI parsing, rejection, JSON success/error shapes, honesty flags.
- Test counts: usage 109, unit 1055.

## JSON CLI Behavior Proof

The following behaviors are verified by existing tests and code inspection:

1. **Reads `/proc/net/dev` exactly once**: `handle_usage_sample(json: true)` calls
   `read_live_proc_net_dev()` which performs a single `std::fs::read_to_string`.
   No loop, no retry, no repeated reads.
   - Verified by: `json_success_with_fake_reader` (single read via injected reader).

2. **Prints JSON only, no human text prefix/suffix**: The JSON path calls
   `serialize_usage_json()` and prints the result directly via `println!()`. No
   human text is prepended or appended.
   - Verified by: `json_output_contains_no_human_header` (phase 8b).

3. **Command exits after one sample**: `handle_usage_sample()` returns `Ok(())`
   after one read-serialize-print cycle. No loop construct exists in the function.
   - Verified by: `cli_remains_single_shot` (structural test).

4. **No loop/watch mode**: The `Usage` variant has only `sample: bool` and
   `json: bool` fields. No `--watch`, `--loop`, or `--interval` flag exists.
   - Verified by: `no_delta_interval_interface_flags_on_usage_command`.

5. **No delta/interval**: No `--delta` or `--interval` flag accepted.
   - Verified by: `no_delta_interval_interface_flags_on_usage_command`.

6. **No interface filter**: No `--interface` flag accepted.
   - Verified by: `no_delta_interval_interface_flags_on_usage_command`.

7. **No arbitrary path input**: Source path hardcoded to `/proc/net/dev`. No
   `--path` or `--source` flag exists.
   - Verified by: `no_arbitrary_path_in_json_mode`, `no_arbitrary_path_argument_exists`.

8. **No sysfs read**: Module source contains no sysfs paths (verified by
   `FORBIDDEN_PATHS` boundary audit in `live_proc_net_dev.rs`).
   - Verified by: boundary audit tests in `tests/live_proc_net_dev/boundary_audit.rs`.

9. **No ledger persistence**: No ledger file read/write in any module involved
   in the JSON output path.
   - Verified by: `honesty.ledger_persistence_performed = false` in JSON output.

10. **No state mutation**: No system state mutation in the JSON output path.
    - Verified by: `honesty.state_mutation_performed = false` in JSON output.

11. **No enforcement/blocking**: No traffic blocking, quota enforcement, limiter
    attach, eBPF, cgroup mutation, PID movement in the JSON output path.
    - Verified by: all corresponding honesty flags `false` in JSON output.

## sampled_at Policy

The `sampled_at` field follows the strict caller-provided-only policy established
in phase 6 and implemented in phase 7:

- **Omitted when `None`**: The `#[serde(skip_serializing_if = "Option::is_none")]`
  attribute ensures the field is completely absent from JSON output. It is not
  serialized as `null`.
- **No silent wall-clock generation**: `handle_usage_sample(json: true)` passes
  `None` to `build_json_from_plan()`, which passes `None` to
  `build_usage_json_from_snapshot()` and `build_usage_json_error()`. No
  `SystemTime::now()` or `Instant::now()` call exists anywhere in the
  production JSON output path.
- **Future extension**: If `--sampled-at "..."` is added in a future phase, the
  field would be set from the caller-provided string. Schema version would need
  to be considered.

## Phase 8b Test Additions

5 optional tests added in `src/commands/usage.rs` to protect the JSON CLI contract:

1. **`json_output_starts_with_brace`**: Asserts JSON output (trimmed) starts with
   `{`, confirming pure JSON with no prefix text.
2. **`json_output_contains_no_human_header`**: Asserts JSON output does not
   contain "zelynic usage --sample", "Read status:", or "interface(s) parsed".
3. **`json_output_contains_warnings_array`**: Asserts warnings array is present
   and non-empty, contains counter-reset and per-app attribution warnings.
4. **`json_error_output_starts_with_brace`**: Asserts error JSON output (trimmed)
   starts with `{`, confirming error responses are also pure JSON.
5. **`json_error_output_contains_no_human_header`**: Asserts error JSON output
   does not contain human text header.

These tests are small additions that directly protect the JSON CLI contract without
changing any Rust behavior.
