# Zelynic v2.8 Phase 5b — Manual Smoke Command Review + Exact Operator Checklist

## Overview

This document provides the exact operator checklist for reviewing the future
first real PID move smoke commands. Phase 5b is **design-only**: it produces
this document and updates related documentation but introduces **no runtime
changes, no PID movement, no `cgroup.procs` writes, no limiter attach, and
no state mutation**.

**Important**: All commands in this document are for **review only**. They
must not be executed until the implementation phase explicitly enables the
real PID move code path. No current Zelynic command performs this move. No
limiter attach is part of this smoke. No nft/tc/Zelynic state mutation is
part of this smoke.

## Phase 5b Boundary

### What Phase 5b Produces

- This operator checklist document
  (`docs/v2.8-phase-5b-manual-smoke-operator-checklist.md`).
- Updates to `docs/experimental-pid-move-lab.md` with phase 5a completed and
  phase 5b current status.
- Updates to `CHANGELOG.md` with a phase 5b entry.
- No Rust code changes. No runtime behavior changes. No new CLI flags.

### What Phase 5b Does NOT Do

- Does NOT implement live PID movement.
- Does NOT write `cgroup.procs`.
- Does NOT execute the smoke commands (only documents them for review).
- Does NOT add runtime mutation of any kind.
- Does NOT add limiter attach, nftables/tc enforcement, or state writes.
- Does NOT change CLI behavior.
- Does NOT modify any Rust source file.

## Operator Checklist: Preflight Environment Checks

Before any smoke command is considered, the operator must verify the following
environment conditions. Each check produces a pass/fail result. If any check
fails, the entire smoke is aborted.

| # | Check | Command | Expected | Abort If |
|---|-------|---------|----------|----------|
| E1 | Running as root | `id -u` | `0` | Not `0` |
| E2 | systemd system bus active | `systemctl is-system-running` | `running` or `degraded` | `offline` or error |
| E3 | cgroup v2 mounted | `mount \| grep 'cgroup2 on /sys/fs/cgroup'` | Contains `rw` | Absent or `ro` |
| E4 | Zelynic namespace writable | `test -w /sys/fs/cgroup/zelynic \|\| mkdir -p /sys/fs/cgroup/zelynic` | Exit 0 | EACCES |
| E5 | No Zelynic processes running | `pgrep -f zelynic \| wc -l` | `0` | Non-zero |
| E6 | nftables baseline clean | `nft list ruleset 2>/dev/null \| grep -ci zelynic` | `0` | Non-zero |
| E7 | tc baseline clean | `tc qdisc show dev lo 2>/dev/null \| grep -ci zelynic` | `0` | Non-zero |

## Operator Checklist: Planned PID Move to Target

This is the **first real write**. After this step, the PID may be in the
target cgroup. If this step fails, no PID was moved and no rollback is needed.

| # | Step | Command | Expected | Abort If |
|---|------|---------|----------|----------|
| W1 | Write PID to target | `echo $TARGET_PID > $TARGET_PATH/cgroup.procs` | Exit 0 | EACCES, ENOENT, ESRCH, EBUSY |
| W2 | Record write result | (capture exit code of W1) | `0` | Non-zero — abort, no rollback needed |

If W1 succeeds, proceed to target verification. If W1 fails, the PID was not
moved. Skip to cleanup (step K1) and stop the sleep scope (step X1).

## Operator Checklist: Planned Target Verification

| # | Step | Command | Expected | Abort If |
|---|------|---------|----------|----------|
| V1 | Read target cgroup.procs | `cat $TARGET_PATH/cgroup.procs` | Line with `$TARGET_PID` | PID not found |
| V2 | Read PID cgroup | `cat /proc/$TARGET_PID/cgroup` | Path contains `zelynic/smoke_target_001` | Does not contain |
| V3 | Record PID location | (document result) | `verified in target` or `unknown` | Ambiguous |

If V2 does not show the PID in the target, record PID location as `unknown`
and proceed to rollback anyway. The PID may have been moved by the kernel
even if verification failed.

## Operator Checklist: Planned Immediate Rollback

Rollback MUST be attempted regardless of whether target verification succeeded.
This is the only rollback attempt. No retry.

| # | Step | Command | Expected | Abort If |
|---|------|---------|----------|----------|
| R1 | Write PID to original | `echo $TARGET_PID > /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs` | Exit 0 | EACCES, ENOENT, ESRCH, EBUSY |
| R2 | Record rollback result | (capture exit code of R1) | `0` | Non-zero — report loudly, proceed to cleanup |

If R1 fails, the PID may be stranded in the target cgroup. For a disposable
`sleep 30`, the PID will exit on its own within seconds. Do not retry.

## Operator Checklist: Planned Rollback Verification

| # | Step | Command | Expected | Abort If |
|---|------|---------|----------|----------|
| B1 | Read original cgroup.procs | `cat /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs \| grep $TARGET_PID` | Line with `$TARGET_PID` | PID not found |
| B2 | Read PID cgroup | `cat /proc/$TARGET_PID/cgroup` | Matches `$ORIGINAL_CGROUP` | Does not match |
| B3 | Record rollback status | (document result) | `verified restored`, `rollback unverified`, or `rollback failed` | — |

If B1 or B2 does not confirm restoration, record as appropriate. The PID may
have exited (for a disposable sleep, this is normal and safe).

## Operator Checklist: Planned Cleanup

| # | Step | Command | Expected | Abort If |
|---|------|---------|----------|----------|
| K1 | Check target is empty | `cat $TARGET_PATH/cgroup.procs` | Empty output | Non-empty — DO NOT remove |

## Explicit Abort Checklist

The smoke MUST be aborted immediately if any of the following conditions is
detected. No `cgroup.procs` write is performed. No PID is moved.

| # | Condition | How to Detect | Action |
|---|-----------|---------------|--------|
| X1 | Not root | `id -u` is not `0` | Abort. Do not proceed. |
| X2 | Not system scope | PID's cgroup is under `/user.slice/` | Abort. User-scope moves are forbidden. |
| X3 | PID missing or stale | `kill -0 $TARGET_PID` fails | Abort. PID has exited. |
| X4 | More than one PID discovered | Target scope has multiple PIDs | Abort. Single PID only. |
| X5 | Original cgroup missing | `test -d /sys/fs/cgroup${ORIGINAL_CGROUP}` fails | Abort. No rollback target. |
| X6 | Original cgroup is Zelynic-managed | Original path contains `zelynic/` | Abort. PID already in Zelynic cgroup. |
| X7 | Target path outside zelynic namespace | Target path does not start with `/sys/fs/cgroup/zelynic/` | Abort. Safety boundary violation. |
| X8 | Target exists and is non-empty | `cat $TARGET_PATH/cgroup.procs` returns PIDs | Abort. Choose different target name. |
| X9 | Cgroup mount is read-only | `mount \| grep cgroup2` shows `ro` | Abort. Cannot write. |
| X10 | Permissions unexpected | `mkdir` or `test -w` fails with EACCES | Abort. Check root context. |
| X11 | Rollback target cannot be verified | Original `cgroup.procs` unreadable/unwritable | Abort. Rollback would be uncertain. |
| X12 | Any output is ambiguous | Unexpected format, extra lines, missing fields | Abort. Do not proceed with unclear data. |

## Expected Observation Table

The operator should observe the following state transitions during the smoke.
Any deviation from these expected observations is a reason to halt and
investigate before proceeding to the next step.

| Phase | Observation | Expected State |
|-------|------------|----------------|
| Before move | `/proc/$TARGET_PID/cgroup` | PID in original cgroup (`$ORIGINAL_CGROUP`) |
| Before move | `$TARGET_PATH/cgroup.procs` | Empty (no PIDs) |
| After target move | `$TARGET_PATH/cgroup.procs` | Contains `$TARGET_PID` |
| After target move | `/proc/$TARGET_PID/cgroup` | Path contains `zelynic/smoke_target_001` |
| After rollback | `/proc/$TARGET_PID/cgroup` | PID restored to `$ORIGINAL_CGROUP` |
| After rollback | Original `cgroup.procs` | Contains `$TARGET_PID` |
| After cleanup | `$TARGET_PATH` | Directory does not exist |
| After cleanup | `/sys/fs/cgroup/zelynic/` | Parent namespace still exists |
| Final | nftables | No Zelynic rules added |
| Final | tc | No Zelynic qdisc/class added |
| Final | Zelynic state | No persistent state written |
| Final | Limiter | No limiter attached |

## Manual Recovery Checklist

If the smoke encounters a failure after the PID move (step W1) succeeded and
rollback or cleanup failed, the operator must follow this recovery checklist
in order.

| # | Step | Command | Notes |
|---|------|---------|-------|
| R1 | Inspect PID cgroup | `cat /proc/$TARGET_PID/cgroup` | If PID has exited, recovery is moot. |
| R2 | Verify original cgroup exists | `test -d /sys/fs/cgroup${ORIGINAL_CGROUP}` | Must exist before any manual rollback. |
| R3 | Verify original cgroup.procs writable | `test -w /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs` | Must be writable. |
| R4 | Move PID back to verified original | `echo $TARGET_PID > /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs` | Only if R2 and R3 pass. |
| R5 | Verify restoration | `cat /proc/$TARGET_PID/cgroup` | Must match `$ORIGINAL_CGROUP`. |
| R6 | Check if target is empty | `cat $TARGET_PATH/cgroup.procs` | If empty, safe to remove. |
| R7 | Remove empty target | `rmdir $TARGET_PATH` | Only if R6 shows empty. |
| R8 | Leave target if non-empty | (do nothing) | Never delete a non-empty cgroup. |
| R9 | Never delete outside zelynic | (verify path starts with `/sys/fs/cgroup/zelynic/`) | Hard safety boundary. |
| R10 | Stop sleep scope | `systemctl stop zelynic-smoke` | Or `kill $TARGET_PID`. |
| R11 | Report PID location | `restored` / `unknown` / `not moved` | Be honest about the actual outcome. |

**Critical recovery rules**:

- Do NOT attempt to write to a cgroup that no longer exists.
- Do NOT create a new cgroup as an alternative restore target.
- Do NOT guess a restore path. Only use the verified original.
- Do NOT delete a non-empty cgroup directory, even if it is
  operation-owned. The unexpected PIDs may belong to another process.
- Do NOT touch any directory outside `/sys/fs/cgroup/zelynic/`.
- If the PID has already exited (disposable sleep), no recovery action
  is needed. The cgroup will be empty when the process exits.

## Output Honesty Requirements

Every smoke execution (in a future phase) must produce output that honestly
describes what happened. The following requirements apply to all output,
whether rendered by Zelynic code or observed manually by the operator.

### Must State

| Field | Required Values | Notes |
|-------|----------------|-------|
| PID moved status | `moved` / `not moved` | Must match actual step W1 outcome |
| PID location after move | `verified in target` / `unknown` | Based on step V1–V3 outcome |
| Rollback attempted | `yes` / `no` | If W1 succeeded, rollback MUST be attempted |
| Rollback outcome | `succeeded` / `failed` / `unverified` | Based on step R1–R3 outcome |
| Target cleanup status | `succeeded` / `failed` / `skipped` | Based on step K1–K3 outcome |

### Must Never Claim

- "Limiter was attached." — No limiter is ever part of this smoke.
- "Bandwidth limiting is active." — No bandwidth limiting is applied.
- "nft/tc/state mutation was performed." — No state mutation occurs.
- "Persistent state was written." — No persistent state is written.
- "PID was restored." (if rollback failed) — Must say "rollback failed".
- "Target was cleaned up." (if rmdir failed) — Must say "cleanup failed".
- "No PID was moved." (if step W1 succeeded) — Must say "PID was moved".

### Canonical Deny Lines

Every output must include the 7 canonical deny lines, with honest
substitutions where real operations occurred:

1. "No live PID move was performed." — Replace with "PID `$TARGET_PID` was
   moved to target cgroup." if step W1 succeeded.
2. "No real cgroup.procs write was performed." — Replace with "cgroup.procs
   write: PID `$TARGET_PID` written to `$TARGET_PATH/cgroup.procs`." if a
   real write occurred.
3. "No limiter attach was performed." — Always present, always true.
4. "No nftables, tc, or Zelynic state changes were made." — Always present,
   always true.
5. "No persistent state write was performed." — Always present, always true.
6. "No CLI path for live PID move was enabled." — Always present, always
   true for manual smoke.
7. "This is fake/model-only output." — Replace with "This is a real manual
   smoke experiment output." if commands were actually executed.
