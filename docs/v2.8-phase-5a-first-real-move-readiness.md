# Zelynic v2.8 Phase 5a — First Real Move Readiness / Manual Smoke Plan

## Overview

This document defines the readiness criteria and manual smoke command plan
for the first real PID move in Zelynic's Experimental PID Move Lab. Phase 5a
is **design-only**: it produces this document and updates related documentation
but introduces **no runtime changes, no PID movement, no `cgroup.procs` writes,
no limiter attach, and no state mutation**.

The first real PID move is a single, disposable, root-only, system-scope-only
cgroup write operation followed by an immediate rollback. No bandwidth
limiting, no nftables/tc enforcement, no persistent state changes, and no
CLI path is enabled. This document specifies the exact manual commands,
abort conditions, expected output honesty requirements, and manual recovery
procedures for that future operation.

## Phase 5a Boundary

### What Phase 5a Produces

- This readiness document
  (`docs/v2.8-phase-5a-first-real-move-readiness.md`).
- Updates to `docs/experimental-pid-move-lab.md` with phase 4e completed and
  phase 5a current status.

### What Phase 5a Does NOT Do

- Does NOT write `cgroup.procs`.
- Does NOT add runtime mutation of any kind.
- Does NOT add limiter attach, nftables/tc enforcement, or state writes.
- Does NOT change CLI behavior.
- Does NOT modify any Rust source file.
- Does NOT execute root smoke commands (only plans them).
- Does NOT write test code.

## Acceptable First Real PID Move Smoke

The only acceptable form for the first real PID move experiment is:

1. **Root-only**: The experiment must be performed as root (UID 0) with
   `CAP_SYS_ADMIN` capability. No unprivileged user context is permitted.

2. **System-scope-only**: The target PID must be a process in the systemd
   system scope (PID 1's cgroup subtree). No user-scope (session) cgroups
   are targeted.

3. **Single disposable sleep PID only**: The target PID must be a freshly
   spawned `sleep` process with a short, bounded lifetime (e.g., `sleep 3` or
   `sleep 5`). The process must have no children, no network sockets, no
   open file descriptors beyond stdin/stdout/stderr, and no shared resources.

4. **Immediate rollback required**: After the move write to the target
   cgroup's `cgroup.procs`, the PID must be written back to the original
   cgroup's `cgroup.procs` immediately — before any other operation, before
   any verification delay, and before any cleanup. The rollback is not optional.

5. **No limiter attach**: The experiment must not attach any bandwidth
   limiter, BPF program, tc class, or any form of enforcement to the target
   cgroup or the moved PID. The move is cgroup-only.

6. **No nftables/tc/Zelynic state mutation**: The experiment must not modify
   any nftables ruleset, tc qdisc/class configuration, or Zelynic persistent
   state file. The move is cgroup-only.

7. **No persistent state write**: The experiment must not write to any file
   outside `/sys/fs/cgroup/zelynic/` or the Zelynic runtime directory (if
   any). No journal entries, no state files, no configuration changes.

8. **No browser/terminal/desktop process**: The target PID must not be a
   browser, terminal emulator, desktop environment component, or any process
   with a graphical user interface.

9. **No user application**: The target PID must not be any user application,
   service, daemon, or process that the user relies on for normal operation.

10. **No multi-PID tree**: The target PID must not have child processes. If
    the sleep process has been reparented to a subreaper or has children at
    smoke time, the experiment is aborted.

11. **No bandwidth limiting claim**: The experiment output must not claim that
    bandwidth limiting is active, that a limiter was attached, or that
    enforcement was applied. The canonical deny lines must still be present.

## Manual Smoke Command Plan

The following command sequence is planned but NOT executed in phase 5a.
This sequence documents the exact steps that a future phase may execute,
provided all abort conditions are satisfied and the operator has reviewed
each command line-by-line.

### Step 1: Create Disposable Sleep Process

```bash
# Run as root. Create a short-lived sleep in a new systemd scope.
systemd-run --unit=zelynic-smoke-test --scope sleep 30
```

The `--unit=zelynic-smoke-test` assigns a known unit name. The `--scope`
creates a transient scope unit. The `sleep 30` gives ample time for the
experiment; the process exits on its own if the experiment fails.

### Step 2: Capture PID

```bash
# Get the main PID of the scope unit.
SYSTEMD_PAGER=cat systemctl show zelynic-smoke-test --property=MainPID --value
```

Record the PID as `$TARGET_PID`. Verify it is a positive integer.

### Step 3: Capture Original Cgroup

```bash
# Read the cgroup v2 path for the target PID.
cat /proc/$TARGET_PID/cgroup
```

Parse the output. Expected format for cgroup v2:

```
0::/user.slice/user-0.slice/session-0.scope/zelynic-smoke-test.scope
```

Or if running under the system slice directly:

```
0::/system.slice/zelynic-smoke-test.scope
```

Record the cgroup path as `$ORIGINAL_CGROUP`. Verify:
- The path starts with `/` and does not end with `/`.
- The path does not contain `zelynic/` as a component (the original cgroup
  must not be Zelynic-managed).

### Step 4: Verify Cgroup Mount

```bash
# Verify the cgroup v2 filesystem is mounted read-write.
mount | grep 'cgroup2 on /sys/fs/cgroup'
```

Expected output: `cgroup2 on /sys/fs/cgroup type cgroup2 (rw,nosuid,nodev,noexec,relatime)`

Abort if the mount is read-only or absent.

### Step 5: Prepare Zelynic Target Cgroup

```bash
# Create the Zelynic namespace if it does not exist.
mkdir -p /sys/fs/cgroup/zelynic

# Create the operation-owned target cgroup.
mkdir /sys/fs/cgroup/zelynic/smoke_target_001
```

Verify the directory was created:

```bash
ls -ld /sys/fs/cgroup/zelynic/smoke_target_001
```

### Step 6: Verify Target Is Empty

```bash
# The target cgroup must be empty before any write.
cat /sys/fs/cgroup/zelynic/smoke_target_001/cgroup.procs
```

Expected output: empty (no PIDs listed). Abort if any PIDs are present.

### Step 7: Planned Move — Write PID to Target

```bash
# Write the PID to the target cgroup's cgroup.procs.
echo $TARGET_PID > /sys/fs/cgroup/zelynic/smoke_target_001/cgroup.procs
```

This is the single, irreversible write that moves the PID. If this succeeds,
the PID is now in the target cgroup.

### Step 8: Verify PID in Target

```bash
# Read the target cgroup's cgroup.procs to confirm the PID arrived.
cat /sys/fs/cgroup/zelynic/smoke_target_001/cgroup.procs
```

Expected output: a single line containing `$TARGET_PID`.

Also verify from the PID's perspective:

```bash
cat /proc/$TARGET_PID/cgroup
```

Expected: the cgroup path now includes `zelynic/smoke_target_001`.

### Step 9: Immediate Rollback — Write PID Back to Original

```bash
# Write the PID back to the original cgroup's cgroup.procs.
echo $TARGET_PID > /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs
```

This is the immediate rollback. It must be attempted regardless of whether
step 8 verification succeeded, and regardless of whether any error occurred
at step 8.

### Step 10: Verify PID Restored

```bash
# Read the original cgroup's cgroup.procs to confirm restoration.
cat /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs | grep $TARGET_PID
```

Also verify from the PID's perspective:

```bash
cat /proc/$TARGET_PID/cgroup
```

Expected: the cgroup path matches `$ORIGINAL_CGROUP`.

### Step 11: Cleanup Target Cgroup

```bash
# Verify the target is empty before removal.
cat /sys/fs/cgroup/zelynic/smoke_target_001/cgroup.procs
```

Expected: empty. If non-empty, do NOT remove — leave it and report.

```bash
# Remove the empty target cgroup.
rmdir /sys/fs/cgroup/zelynic/smoke_target_001
```

### Step 12: Verify No Leftover Target

```bash
# Confirm the target directory no longer exists.
ls /sys/fs/cgroup/zelynic/smoke_target_001 2>&1
```

Expected: `ls: cannot access '/sys/fs/cgroup/zelynic/smoke_target_001': No such file or directory`

### Step 13: Verify No nft/tc/State Changes

```bash
# Verify nftables rulesets are unchanged (no Zelynic rules added).
nft list ruleset 2>/dev/null | grep -i zelynic
```

Expected: no output (no Zelynic rules).

```bash
# Verify tc configuration is unchanged on the interface.
tc qdisc show dev lo 2>/dev/null | grep -i zelynic
```

Expected: no output (no Zelynic qdisc/class).

### Step 14: Stop Disposable Sleep Scope

```bash
# Stop the sleep scope now that the experiment is complete.
systemctl stop zelynic-smoke-test
```

The sleep process exits. Cleanup is complete.

## Abort Conditions

The experiment MUST be aborted immediately (no write to `cgroup.procs`, no
PID move) if any of the following conditions is detected before step 7:

1. **PID missing or stale**: The `MainPID` is 0, empty, or the process at
   `/proc/$TARGET_PID` no longer exists. Check with `kill -0 $TARGET_PID`.

2. **More than one PID discovered**: The target scope contains more than one
   process, or `cat /proc/$TARGET_PID/cgroup` reveals an unexpected process
   tree under the scope.

3. **Original cgroup missing**: The cgroup path from step 3 does not
   correspond to a valid directory under `/sys/fs/cgroup`. Check with
   `test -d /sys/fs/cgroup${ORIGINAL_CGROUP}`.

4. **Original cgroup is Zelynic-managed**: The original cgroup path contains
   `zelynic/` as a path component. The PID is already in a Zelynic-managed
   cgroup and must not be moved again.

5. **Target path outside `/sys/fs/cgroup/zelynic/`**: The target cgroup
   directory would be created outside the Zelynic namespace. This must
   never happen.

6. **Target already exists and is non-empty**: If
   `/sys/fs/cgroup/zelynic/smoke_target_001` already exists and contains
   PIDs, it must not be used. Choose a different target name or remove it
   only if it is empty and operation-owned.

7. **Cgroup mount is read-only**: The cgroup2 filesystem at
   `/sys/fs/cgroup` is mounted read-only. No writes are possible. Abort.

8. **Permissions unexpected**: The target directory creation fails with
   EACCES, or the `cgroup.procs` write test (dry run) fails. Verify root
   context with `id`.

9. **Rollback target cannot be verified**: If the original cgroup directory
   exists but its `cgroup.procs` file cannot be read or written (permission
   issue), abort. The rollback would be uncertain.

10. **Any command output is ambiguous**: If any step produces unexpected
    output — wrong format, extra lines, missing expected fields — abort.
    Do not proceed with a write if the pre-flight information is unclear.

## Expected Output Honesty

Every execution of the smoke experiment (in a future phase) must produce
output that honestly describes what happened. The following requirements
apply to all output:

### Must State

| Field | Required Values | Notes |
|-------|----------------|-------|
| PID moved status | `moved` / `not moved` | Must match actual outcome |
| PID location after move | `verified in target` / `unknown` | Based on step 8 verification |
| Rollback attempted | `yes` / `no` | If step 7 succeeded, rollback MUST be attempted |
| Rollback outcome | `succeeded` / `failed` / `unverified` | Based on step 10 verification |
| Target cleanup status | `succeeded` / `failed` / `skipped` | Based on step 11 outcome |
| Original cgroup | `<path>` | The actual path from step 3 |

### Must NOT Claim

- "Limiter was attached." — No limiter is ever attached in this experiment.
- "Bandwidth limiting is active." — No bandwidth limiting is applied.
- "nft/tc/state mutation was performed." — No state mutation occurs.
- "Persistent state was written." — No persistent state is written.
- "PID was restored." (if rollback failed) — Must say "rollback failed" instead.
- "Target was cleaned up." (if rmdir failed) — Must say "cleanup failed" instead.

### Canonical Deny Lines

Every output must include the 7 canonical deny lines from the failure
simulation subsystem (phase 4d):

1. "No live PID move was performed." — Replace with honest statement if PID
   was actually moved (e.g., "PID <pid> was moved to target cgroup.").
2. "No real cgroup.procs write was performed." — Replace with honest
   statement if a real write occurred (e.g., "cgroup.procs write: PID <pid>
   written to <target>/cgroup.procs.").
3. "No limiter attach was performed." — Always true.
4. "No nftables, tc, or Zelynic state changes were made." — Always true.
5. "No persistent state write was performed." — Always true.
6. "No CLI path for live PID move was enabled." — Always true.
7. "This is fake/model-only output." — Replace with "This is a real smoke
   experiment output." if the commands were actually executed.

## Manual Recovery

If the smoke experiment encounters a failure after step 7 (the PID was moved
but rollback or cleanup failed), the following recovery procedures apply.

### Inspect PID Current Cgroup

```bash
# Check where the PID currently resides.
cat /proc/$TARGET_PID/cgroup
```

If the PID has exited (process is dead), the question is moot. For a
disposable `sleep 30`, the PID will exit on its own within 30 seconds.

### Move PID Back (Only If Original Cgroup Is Verified)

If the PID is alive and stranded in the Zelynic target cgroup, and the
original cgroup path is verified to exist and be writable:

```bash
# Verify the original cgroup still exists.
test -d /sys/fs/cgroup${ORIGINAL_CGROUP} && echo "OK" || echo "MISSING"

# Verify the original cgroup.procs is writable.
test -w /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs && echo "OK" || echo "NO_WRITE"

# Move PID back to original cgroup.
echo $TARGET_PID > /sys/fs/cgroup${ORIGINAL_CGROUP}/cgroup.procs

# Verify restoration.
cat /proc/$TARGET_PID/cgroup
```

Do NOT attempt to write to a cgroup that no longer exists. Do NOT create a
new cgroup as an alternative restore target. Do NOT guess a restore path.

### Leave Target If Non-Empty

If the target cgroup `/sys/fs/cgroup/zelynic/smoke_target_001` is non-empty
after rollback, do NOT remove it. A non-empty cgroup directory indicates
that other PIDs or processes are present, and removing it would strand them.

```bash
# Check if target is empty before attempting removal.
cat /sys/fs/cgroup/zelynic/smoke_target_001/cgroup.procs
```

If empty: safe to `rmdir`. If non-empty: leave in place and report.

### Avoid Deleting Outside `/sys/fs/cgroup/zelynic/`

No recovery command must touch any directory outside
`/sys/fs/cgroup/zelynic/`. The original cgroup, the system root cgroup,
and all other system cgroups are never modified or removed. Only
operation-owned directories under the Zelynic namespace are eligible for
cleanup.

### Stop Disposable Sleep Scope/Process

If the sleep process is still running and the experiment is complete (or
aborted):

```bash
# Stop the scope unit.
systemctl stop zelynic-smoke-test

# Or kill the process directly if the scope unit failed.
kill $TARGET_PID 2>/dev/null
```

For a `sleep 30` process, this is always safe — the process has no state,
no children, and no resources to clean up.

The failure simulation subsystem (phases 4a–4e) provides the theoretical
foundation for this smoke plan:

- **Phase 4a design** (12 failure scenarios F1–F12) maps directly to the
  abort conditions and recovery procedures defined here.

  every failure point in the transaction.

  requirements and canonical deny lines.

  entry criteria that this document fulfills.

model from phase 3a, with steps mapped as follows:

| 3 | — | Capture original cgroup |
