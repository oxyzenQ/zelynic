# v3.1 Phase 4: Pure Ledger Identity Report Model

## Status

**Phase 4 — Pure report model only. No CLI wiring. No live process scanning. No persistence writes. No JSON schema change.**

## Summary

Phase 4 creates a pure report model that consumes existing `LedgerIdentityAttachment`
values (from phase 3) and produces deterministic report structures with summaries by
interface, attribution scope, and identity type. This is a pure model/render phase —
no CLI wiring, no live system reads, no persistence writes, no enforcement fields
become active, and the existing v3.0 usage JSON schema remains unchanged. The report
model does NOT replace the existing `ledger_inspect` module; it is a separate additive
module that consumes phase 3 attachment values.

## Design Decision

### Chosen Approach: Small Additive Report Module

A new `src/accounting/ledger_identity_report.rs` module that consumes
`LedgerIdentityAttachment` values and produces deterministic report structures.
This approach was chosen because:

1. **Consumes existing models.** The report module imports `LedgerIdentityAttachment`
   from phase 3 and `UsageAttributionScope` from phase 2. It does not re-define any
   existing types or duplicate the ledger system.

2. **Does not replace existing modules.** The report model is separate from
   `ledger_inspect` (which operates on `Ledger` directly). Both modules coexist
   without conflict.

3. **Deterministic ordering via BTreeMap.** All grouping and sorting uses
   `BTreeMap` for consistent, deterministic output regardless of input order.

4. **Saturating arithmetic.** All byte totals use saturating addition to avoid
   overflow panics with large counter values.

5. **JSON-serializable.** All report types derive `Serialize`/`Deserialize` using
   existing serde dependency — no new dependencies needed.

## Model Types Added

| Type | Module | Description |
|------|--------|-------------|
| `LedgerIdentityReport` | `src/accounting/ledger_identity_report.rs` | Top-level report: totals, per-target breakdown, per-interface breakdown, scope counts, honesty flags, provenance. |
| `LedgerIdentityReportTarget` | `src/accounting/ledger_identity_report.rs` | Per-target summary: deterministic key, interface, attribution scope, optional process/cgroup identity hints, byte totals, entry count. |
| `LedgerIdentityReportInterface` | `src/accounting/ledger_identity_report.rs` | Per-interface aggregate: interface name, loopback flag, byte totals, entry count. |
| `LedgerIdentityReportTotals` | `src/accounting/ledger_identity_report.rs` | Aggregate byte totals: total_rx_bytes, total_tx_bytes, total_combined_bytes, entry_count. |
| `LedgerIdentityReportHonesty` | `src/accounting/ledger_identity_report.rs` | Report-level honesty and safety flags (14 fields). All enforcement/persistence/mutation flags are hard-coded `false`. Attribution is always best-effort. |
| `build_ledger_identity_report(attachments)` | `src/accounting/ledger_identity_report.rs` | Builds deterministic report from a slice of `LedgerIdentityAttachment` values. Groups by interface + scope + identity hints using BTreeMap. |
| `default_report_honesty()` | `src/accounting/ledger_identity_report.rs` | Returns default honesty flags with all enforcement inactive. |
| `serialize_report_json(report)` | `src/accounting/ledger_identity_report.rs` | Serializes report to deterministic pretty JSON. |
| `deserialize_report_json(json)` | `src/accounting/ledger_identity_report.rs` | Deserializes report from JSON. |
| `render_ledger_identity_report(report)` | `src/accounting/ledger_identity_report.rs` | Human-readable render with scope counts, interface breakdown, target breakdown, totals, and 13 safety disclaimers. |

## Required Behavior (All Met)

1. Builds a deterministic report from a slice of `LedgerIdentityAttachment` values.
2. Summarizes by interface.
3. Summarizes by attribution scope.
4. Summarizes optional process identity (comm, PID).
5. Summarizes optional cgroup identity (cgroup_path, systemd_unit).
6. Summarizes unknown target entries.
7. Preserves total rx/tx/combined bytes.
8. Preserves entry counts.
9. Preserves interface-level honesty.
10. Never claims exact per-app attribution.
11. Marks process/cgroup/target attribution as best-effort.
12. Keeps enforcement inactive.
13. Keeps persistence false.
14. Keeps filesystem_write false.
15. Keeps network_blocking false.
16. Keeps quota false.
17. Keeps eBPF false.
18. Keeps nft/tc mutation false.
19. Keeps cgroup mutation false.
20. Keeps daemon/watch false.
