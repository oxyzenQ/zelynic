# Zelynic v3.0 Phase 6: JSON Output Contract Design for `usage --sample --json`

This document defines the JSON output contract for a future `zelynic usage --sample
--json` command. Phase 6 is **design-only**: it produces this document and updates
related documentation but does not enable `--json`, does not implement JSON
serialization, does not modify the CLI, does not add Rust code changes, and does not
introduce any runtime behavior changes.

## Purpose

Phase 6 designs the structured JSON output schema that will be emitted when
`zelynic usage --sample --json` is invoked in a future phase. The contract extends
the existing human-readable output contract (13 honesty disclaimers) into a
machine-readable format suitable for scripting, monitoring integrations, and dashboard
feeds. The design preserves all safety boundaries from previous phases and adds
explicit constraints for JSON-specific concerns such as timestamp honesty and error
representation.

This phase does NOT implement `--json`. The flag is not wired into the CLI parser.
No JSON serialization code is added. No model types for JSON output are defined in
Rust. The contract is purely a design document that constrains future implementation.

## JSON Output Schema

### Success Response

The JSON output for `zelynic usage --sample --json` follows this schema:

```json
{
  "schema_version": 1,
  "command": "usage --sample --json",
  "source_path": "/proc/net/dev",
  "source_label": "live_proc_net_dev",
  "sampled_at": null,
  "interfaces": [
    {
      "name": "wlan0",
      "rx_bytes": 1320489368,
      "tx_bytes": 358637568,
      "combined_bytes": 1679126936,
      "rx_packets": 1234567,
      "tx_packets": 345678,
      "loopback": false
    },
    {
      "name": "lo",
      "rx_bytes": 524288,
      "tx_bytes": 524288,
      "combined_bytes": 1048576,
      "rx_packets": 100,
      "tx_packets": 200,
      "loopback": true
    }
  ],
  "totals": {
    "total_rx_bytes": 1321013656,
    "total_tx_bytes": 359162856,
    "total_combined_bytes": 1680176512,
    "interface_count": 2
  },
  "honesty": {
    "interface_level_only": true,
    "per_app_attribution": false,
    "quota_enforcement_active": false,
    "network_blocking_active": false,
    "limiter_attach_performed": false,
    "nft_tc_state_mutation_performed": false,
    "ledger_persistence_performed": false,
    "ebpf_used": false,
    "cgroup_mutation_performed": false,
    "pid_movement_performed": false,
    "filesystem_write_performed": false,
    "state_mutation_performed": false
  },
  "warnings": [
    "Counters may reset after reboot or interface reset.",
    "Not per-app attribution: interface counters are aggregate per-interface."
  ]
}
```

### Field Definitions

#### Top-Level Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `schema_version` | integer | Yes | Schema version for forward compatibility. Currently `1`. Must be the first field. |
| `command` | string | Yes | The command that produced this output. Always `"usage --sample --json"`. |
| `source_path` | string | Yes | Filesystem path of the data source. Always `"/proc/net/dev"`. |
| `source_label` | string | Yes | Source identifier. `"live_proc_net_dev"` for live reads, `"live_proc_net_dev_sample"` for injected test content. |
| `sampled_at` | string or null | Conditional | ISO 8601 timestamp if explicitly provided by caller. `null` if omitted. See sampled_at policy below. |
| `interfaces` | array | Yes | Per-interface counter data. May be empty if no interfaces are parsed. |
| `totals` | object | Yes | Aggregate totals across all parsed interfaces. |
| `honesty` | object | Yes | Honesty flags. Present in every JSON response including errors. |
| `warnings` | array of strings | Yes | Warning messages. Always includes counter-reset and attribution warnings. |

#### Per-Interface Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Interface name as parsed from `/proc/net/dev` (e.g., `wlan0`, `eth0`, `lo`). |
| `rx_bytes` | integer | Yes | Cumulative received bytes since boot or interface-up. `u64` range. |
| `tx_bytes` | integer | Yes | Cumulative transmitted bytes since boot or interface-up. `u64` range. |
| `combined_bytes` | integer | Yes | `rx_bytes + tx_bytes`. Saturating addition for overflow safety. |
| `rx_packets` | integer | Conditional | Cumulative received packets. Present if parsed from source. `0` if not available. |
| `tx_packets` | integer | Conditional | Cumulative transmitted packets. Present if parsed from source. `0` if not available. |
| `loopback` | boolean | Yes | `true` if this is the loopback interface (`lo`). |

#### Totals Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `total_rx_bytes` | integer | Yes | Sum of all interface RX bytes (saturating). |
| `total_tx_bytes` | integer | Yes | Sum of all interface TX bytes (saturating). |
| `total_combined_bytes` | integer | Yes | Sum of all interface combined bytes (saturating). |
| `interface_count` | integer | Yes | Number of interfaces in the `interfaces` array. |

#### Honesty Flags

All honesty flags are boolean constants in the current implementation. These flags
must never be `true` in v3.0. Future versions that implement per-app attribution
(via eBPF or other means) must increment `schema_version` and update flag
documentation accordingly.

| Field | Type | Value in v3.0 | Description |
|-------|------|---------------|-------------|
| `interface_level_only` | boolean | `true` | Data is interface-level aggregate counters. Always `true` in v3.0. |
| `per_app_attribution` | boolean | `false` | Per-process/per-app traffic attribution is not available. |
| `quota_enforcement_active` | boolean | `false` | No quota enforcement is implemented or active. |
| `network_blocking_active` | boolean | `false` | No network traffic is blocked, dropped, or rejected. |
| `limiter_attach_performed` | boolean | `false` | No rate limiter is attached. |
| `nft_tc_state_mutation_performed` | boolean | `false` | No nftables rules, tc qdisc/filters/classes, or Zelynic runtime state is modified. |
| `ledger_persistence_performed` | boolean | `false` | No ledger file is read or written. |
| `ebpf_used` | boolean | `false` | No eBPF programs are loaded or attached. |
| `cgroup_mutation_performed` | boolean | `false` | No cgroup creation, modification, or `cgroup.procs` write is performed. |
| `pid_movement_performed` | boolean | `false` | No PIDs are moved between cgroups. |
| `filesystem_write_performed` | boolean | `false` | No filesystem write is performed. |
| `state_mutation_performed` | boolean | `false` | No system state mutation of any kind is performed. |

These 12 honesty flags extend the 13 human-readable honesty lines from the text
output. The mapping is:

- `interface_level_only` maps to "interface-level only (not per-app attribution)"
- `per_app_attribution` maps to "not per-app attribution" (inverted: false means not)
- `quota_enforcement_active` maps to "no quota enforcement active"
- `network_blocking_active` maps to "no network blocking active"
- `limiter_attach_performed` maps to "no limiter attach performed"
- `nft_tc_state_mutation_performed` maps to "no nft/tc/Zelynic state mutation performed"
- `ledger_persistence_performed` maps to "no ledger persistence performed"
- `ebpf_used` maps to "no eBPF used"
- `cgroup_mutation_performed` maps to "no cgroup mutation"
- `pid_movement_performed` maps to "no PID movement"
- `filesystem_write_performed` maps to "filesystem write not performed" (inverted)
- `state_mutation_performed` maps to "state mutation not performed" (inverted)

The remaining human-readable lines ("read-only /proc/net/dev seam" and "counters
may reset after reboot/interface reset") are covered by `source_path`/`source_label`
and the `warnings` array respectively.

#### Warning Messages

| Warning | Always Present | Description |
|---------|---------------|-------------|
| "Counters may reset after reboot or interface reset." | Yes | Cumulative counters reset on reboot or interface down/up. |
| "Not per-app attribution: interface counters are aggregate per-interface." | Yes | Data does not identify which process sent or received which bytes. |

Warnings may be extended in future schema versions. Consumers should tolerate
unknown warnings.

### sampled_at Field Policy

The `sampled_at` field is **only populated if the caller explicitly provides a
timestamp**. The command does not silently generate a wall-clock timestamp.

This policy prevents misleading precision in JSON output where the user did not
request a timestamped sample. It also prevents model tests from silently
generating timestamps that are not meaningful in a test context.

**Allowed patterns:**

| Invocation | `sampled_at` value |
|------------|-------------------|
| `zelynic usage --sample --json` | `null` (omitted or explicitly null) |
| `zelynic usage --sample --json --sampled-at "2026-06-07T12:00:00Z"` | `"2026-06-07T12:00:00Z"` (caller-provided) |

**Forbidden patterns:**

| Pattern | Reason |
|---------|--------|
| Silently generating `SystemTime::now()` | Misleading precision; not requested by user |
| Generating timestamps in model/serialization tests | Tests should not produce fake wall-clock times |
| Using `Instant::now()` for display | Not a wall-clock time; confusing in JSON |

**Implementation constraint for future phases:** When `--sampled-at` is not
provided by the caller, the field must be `null` or omitted entirely from the JSON
output. The serialization code must not insert a default timestamp.

## Error JSON Contract

Error responses must follow the same schema structure with error-specific fields
replacing the normal data. All errors retain honesty flags and warnings.

### read_error

Returned when `/proc/net/dev` cannot be read (file not found, permission denied,
I/O error).

```json
{
  "schema_version": 1,
  "command": "usage --sample --json",
  "source_path": "/proc/net/dev",
  "source_label": "live_proc_net_dev",
  "sampled_at": null,
  "error": {
    "type": "read_error",
    "message": "read error: permission denied"
  },
  "interfaces": [],
  "totals": {
    "total_rx_bytes": 0,
    "total_tx_bytes": 0,
    "total_combined_bytes": 0,
    "interface_count": 0
  },
  "honesty": {
    "interface_level_only": true,
    "per_app_attribution": false,
    "quota_enforcement_active": false,
    "network_blocking_active": false,
    "limiter_attach_performed": false,
    "nft_tc_state_mutation_performed": false,
    "ledger_persistence_performed": false,
    "ebpf_used": false,
    "cgroup_mutation_performed": false,
    "pid_movement_performed": false,

```

### parse_error

Returned when `/proc/net/dev` is read successfully but the content cannot be
parsed (malformed format, unexpected header, non-numeric counter values).

```json
{
  "schema_version": 1,
  "command": "usage --sample --json",
  "source_path": "/proc/net/dev",
  "source_label": "live_proc_net_dev",
  "sampled_at": null,
  "error": {
    "type": "parse_error",
    "message": "parse error: missing colon after interface name"
  },
  "interfaces": [],
  "totals": {
    "total_rx_bytes": 0,
    "total_tx_bytes": 0,
    "total_combined_bytes": 0,
    "interface_count": 0
  },
  "honesty": { ... },
  "warnings": [ ... ]
}
```

### unsupported_flag_error

Returned when a flag combination is not supported in the current phase.

```json
{
  "schema_version": 1,
  "command": "usage --sample --json",
  "source_path": "/proc/net/dev",
  "source_label": "live_proc_net_dev",
  "sampled_at": null,
  "error": {
    "type": "unsupported_flag_error",
    "message": "unsupported flag: --delta is not implemented"
  },
  "interfaces": [],
  "totals": {
    "total_rx_bytes": 0,
    "total_tx_bytes": 0,
    "total_combined_bytes": 0,
    "interface_count": 0
  },
  "honesty": { ... },
  "warnings": [ ... ]
}
```

### Error Contract Rules

1. **All errors retain honesty flags.** The `honesty` object is always present
   with the same 12 boolean fields, regardless of whether the response is a
   success or an error.
2. **Errors must not claim partial enforcement or mutation.** Even in error
   responses, all honesty flags must remain their v3.0 constant values
   (`false` for all mutation/enforcement flags, `true` for `interface_level_only`).
3. **The `error` object is mutually exclusive with interface data.** When
   `error` is present, `interfaces` is an empty array and `totals` are all zero.
   When `error` is absent, `interfaces` contains the parsed data.
4. **Error messages are human-readable strings.** They are not structured error
   codes. The `type` field identifies the error category for programmatic handling.
5. **No error response implies enforcement, blocking, or mutation.** Error
   responses must never contain `quota_enforcement_active: true` or any other
   positive mutation flag.

## CLI Behavior Contract

### `usage --sample --json` (Future Command)

The full future command: `zelynic usage --sample --json`. This reads `/proc/net/dev`
exactly once, parses the content, serializes to JSON, and prints to stdout. The
JSON output follows the schema defined above. The command exits with code 0 on
success, code 1 on error (read failure, parse failure).

### `usage --json` without `--sample` Must Not Read `/proc/net/dev`

The `--json` flag requires `--sample`. Invoking `zelynic usage --json` without
`--sample` must either be rejected by clap (if `--json` is defined on the `Usage`
variant and requires `--sample`) or must not perform any live filesystem read.

**Contract:** `--json` without `--sample` must not trigger a read of `/proc/net/dev`.
This is enforced by making `--json` require `--sample` in clap or by checking both
flags in the dispatch handler.

### Single-Shot Only

`usage --sample --json` remains single-shot. One read, one parse, one JSON output,
exit. No loop, no watch, no polling, no interval, no retry.

**Contract:** The command must not contain any loop construct, background thread,
timer, or polling mechanism. It reads once, serializes once, prints once, exits.

### Prohibited Features

The following features must not be enabled by the JSON output contract:

| Feature | Status | Reason |
|---------|--------|--------|
| Loop/watch mode | Hard-blocked | JSON output is single-shot only |
| Delta computation | Hard-blocked | Requires two reads; deferred to future phase |
| Interval sampling | Hard-blocked | Requires repeated reads; deferred to future phase |
| Arbitrary path input | Hard-blocked | Source path is hardcoded to `/proc/net/dev` |
| Interface filtering | Hard-blocked in v3.0 | May be added in a future explicit phase |
| Persistence | Hard-blocked | No ledger file read/write in v3.0 |
| eBPF | Hard-blocked | Not implemented in v3.0 |
| Quota enforcement | Hard-blocked | Not implemented in v3.0 |
| Network blocking | Hard-blocked | Not implemented in v3.0 |

### Interface Filtering Note

Interface filtering (`--interface wlan0`) is not included in the v3.0 JSON output
contract. A future phase may add interface filtering to both text and JSON output,
but this requires an explicit design phase with its own gate criteria. The v3.0
JSON output always includes all interfaces parsed from `/proc/net/dev`.
